## Bug 1: EXIT/TRIM 死区 — locked win 保护缺少 margin 检查

### 问题

EXIT Layer 1 和 TRIM 的 locked-win 保护仅检查 `wu_round(daily_max) > slot.upper`，
不要求 margin≥2。而 `evaluate_locked_win_signals` 入场时要求
`(wu_round(daily_max) - upper) >= locked_win_margin_f`（默认 2）才算 locked win。

这导致一个**死区**：当 daily_max 刚超出 slot upper 但差值 < margin 时，
- locked_win_signals 不会触发（差值不够）
- EXIT Layer 1 / TRIM 却认为"已锁定胜利"拒绝退出
- 结果：持仓被锁死，无法退出也无法获得 locked win 保护

**示例：** daily_max=73.2°F, slot upper=72°F, margin=2
- `wu_round(73.2) = 73`, gap = 73 - 72 = 1 < 2
- locked_win 不触发 ✗
- 旧逻辑：EXIT/TRIM 跳过（73 > 72）→ 死区 ✗

### 修复

- `evaluator.py` EXIT Layer 1 (原590行): 改为检查 `gap >= config.locked_win_margin_f`
- `evaluator.py` TRIM 保护 (原353行): 同上

### 影响范围

- 仅影响 daily_max 超出 slot upper 且差值在 [1, margin) 范围的边缘情况
- 真正的 locked win（差值 ≥ margin）行为不变
- evaluate_locked_win_signals 不受影响

### 测试变更

- `test_hybrid_exit.py::TestLayer1LockedWinProtection::test_locked_win_not_exited` — daily_max 改为 87（gap=3 ≥ 2）
- `test_temperature_boundaries.py::TestExitWuRound::test_exit_protected_when_wu_round_exceeds` — daily_max 改为 86.5（gap=3 ≥ 2）
- `test_temperature_boundaries.py::TestTrimWuRound::test_trim_protected_when_wu_round_exceeds` — daily_max 改为 76.5（gap=3 ≥ 2）
- 新增死区专用测试（断言 `len(sigs) == 1`，用 forecast-inside-slot 迫使负 EV 退出，验证 Layer 1 确实没有阻止）：
  - `test_hybrid_exit.py::test_locked_win_dead_zone_not_protected` — gap=1, forecast=82 inside [80,84]
  - `test_temperature_boundaries.py::test_exit_dead_zone_not_protected` — gap=1, forecast=82 inside [80,84]
  - `test_temperature_boundaries.py::test_trim_dead_zone_not_protected` — gap=1, forecast=72 inside [70,74]

## Bug 2: 缺少 min_no_price 下限

### 问题

NO 价格仅有上限 (`max_no_price=0.85`)，没有下限。极低价格（如 $0.004）
能通过筛选进入交易信号。

极低价问题：
1. 流动性极差，挂单几乎无法成交
2. `net_odds = (1 - price) / price` 在极低价时爆炸式放大，Kelly sizing 失真
3. EV 公式中几乎任何 win_prob 都能产生正 EV，失去过滤意义

### 修复

- `config.py` StrategyConfig: 新增 `min_no_price: float = 0.20`
- `evaluator.py` evaluate_no_signals: 在 `price_no > max_no_price` 检查前新增 `price_no < config.min_no_price` 过滤
- `evaluator.py` evaluate_locked_win_signals: 同样新增下限检查（locked win 也应有流动性门槛）
- TRIM/EXIT 不添加（已持有仓位，入场时已验证过价格）

### 影响范围

- 过滤掉 NO 价格 < $0.20 的所有入场信号（含 locked win）
- 不影响已持有仓位的退出/修剪逻辑
- 策略变体如需自定义下限，可在 `get_strategy_variants()` 中覆盖

### 测试变更

- `test_locked_win.py::test_price_no_very_low_high_ev` → 重命名为 `test_price_no_very_low_filtered`，断言改为 0 信号
- 新增 `test_price_no_at_min_accepted`: price_no=0.20 恰好等于下限，应被接受

## Bug 3: daily_max 在 "below X°F" slot 内仍允许买 NO（post-peak）

### 问题

入场逻辑已有两类 daily_max in-slot 检查：
- "≥X°F" slot: `wu_round(daily_max) >= X` → 阻止买 NO ✓ (第193-200行)
- Range slot [L,U]: `L <= wu_round(daily_max) <= U` → 阻止买 NO ✓ (第205-212行)

但**缺少** "below X°F" slot（`lower=None, upper=X`）的检查：
- 当 `wu_round(daily_max) < X` 时，daily_max 在 slot 范围 `(-∞, X)` 内
- YES 可能赢 → NO 是输家 → 不应买 NO
- 旧逻辑没有阻止 → 错误入场

### 修复

- `evaluator.py` evaluate_no_signals (原212行后): 新增 post-peak 检查
- 仅在 `peak_conf is not None`（local_hour ≥ 14）时生效，避免早间误判
- 条件: `slot.temp_lower_f is None and slot.temp_upper_f is not None and wu_round(daily_max_f) < int(slot.temp_upper_f)`

### 为什么限制在 post-peak

早间 daily_max 低是正常的（还没到峰值）。如 daily_max=50°F, slot="below 75°F"，
此时温度可能还会大幅上升使 NO 变为赢家。post-peak 后 daily_max 接近最终值，
此时仍在 slot 内说明 YES 确实可能赢。

### 影响范围

- 仅影响 "below X°F" 类型 slot 在 post-peak 时段的入场
- Range slot 和 "≥X°F" slot 不受影响
- locked_win_signals / exit / trim 不受影响

### 边界情况

- `wu_round(daily_max) == int(upper)`: 不阻止（`<` 不含等于），此时 daily_max 恰好在边界，NO 仍有可能
- `local_hour = 14`（peak_conf 生效但温度可能还涨 ±3°F）: 已由 `_post_peak_confidence` 返回较大置信区间处理
- `days_ahead > 0`: daily_max_f 为 None，自然跳过

### 测试（`test_entry_guards.py::TestBelowSlotDailyMaxInSlot`）

- `test_post_peak_daily_max_inside_slot_blocked` — hour=18, daily_max=70, slot=(None,75) → 0 signals
- `test_post_peak_daily_max_above_slot_allowed` — hour=18, daily_max=76, slot=(None,75) → 1 signal
- `test_pre_peak_daily_max_inside_slot_allowed` — hour=12, daily_max=70, slot=(None,75) → 1 signal
- `test_boundary_wu_round_equals_upper_allowed` — hour=18, daily_max=75.4 (wu_round=75==upper), slot=(None,75) → 1 signal
